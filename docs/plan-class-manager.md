# Class Manager 기획서

> 작성일: 2026-04-29

## 1. 프로젝트 개요

대학교와 기업 등 여러 기관에서 강의하는 강사가 모든 수강생을 하나의 플랫폼에서 통합 관리할 수 있는 수업 관리 도구. 기관별로 분산된 수강생 정보를 한곳에 모아 출결, 공지, 자료 배포 등을 단순하고 빠르게 처리하는 것을 목표로 한다.

## 2. 문제 정의

- 대학교 강의와 기업 강의를 동시에 진행하면 수강생 명단·연락처·진도가 제각각 다른 곳에 흩어짐
- 기존 LMS(e-Class, Canvas 등)는 기관별로 따로 존재해 강사가 매번 다른 시스템에 로그인해야 함
- 엑셀·구글 시트로 관리하면 공유·업데이트가 번거롭고 실수가 잦음

## 3. 타겟 사용자

| 역할 | 설명 |
|------|------|
| 강사 (어드민) | 대학·기업 등 복수 기관에서 강의하는 1인 강사 |
| 수강생 | 대학생 또는 기업 임직원, 강사가 초대하여 등록 |

**핵심 니즈**
- 강사: 기관별 수강생을 한눈에 보고 싶다
- 강사: 공지·자료를 수업별로 빠르게 올리고 싶다
- 수강생: 내 수업 자료와 공지를 한 곳에서 확인하고 싶다

## 4. 핵심 기능 (MVP)

| 기능 | 설명 | 우선순위 |
|------|------|---------|
| 수업(클래스) 생성·관리 | 기관명, 과목명, 기간으로 수업 단위를 생성·수정·삭제 | 높음 |
| 수강생 등록·관리 | 수업별 수강생 초대(이메일), 목록 조회, 상태 관리 | 높음 |
| 강사/수강생 인증 | 이메일+비밀번호 로그인, 역할(강사/수강생) 분리 | 높음 |
| 공지·자료 게시 | 수업별 공지사항 작성 및 첨부파일·링크 업로드 | 중간 |
| 대시보드 | 강사: 전체 수업 현황 요약 / 수강생: 내 수업 목록 | 중간 |
| 출결 체크 (선택) | 수업 회차별 출석 여부 기록 | 낮음 |

## 5. 차별화 포인트

- **기관 통합 관리**: 대학·기업 등 소속과 무관하게 수강생을 하나의 화면에서 관리
- **강사 1인 최적화**: 복잡한 학사 기능 없이 강사 혼자 빠르게 운영할 수 있는 단순한 UX
- **즉시 사용 가능**: 별도 설치 없이 웹 접속만으로 시작

## 6. 비즈니스 목표

- 강사 본인이 직접 사용하는 실사용 포트폴리오 프로젝트
- 추후 동일한 상황(복수 기관 강사)의 사용자에게 확장 가능한 SaaS 씨앗 프로젝트

## 7. 제약사항 및 리스크

| 구분 | 내용 |
|------|------|
| 일정 | 1개월 이내 MVP 출시 목표 — 범위를 최소화해야 함 |
| 팀 | 1인 풀스택 개발 — 복잡한 인프라 설정은 피하고 관리형 서비스 활용 권장 |
| 예산 | 무료 또는 저비용 인프라 (Vercel, Supabase 등 무료 티어) |
| 리스크 | 개인정보(수강생 이메일) 취급 → 최소한의 데이터만 수집하고 보안 기본기 준수 필요 |
